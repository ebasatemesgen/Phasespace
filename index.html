System Overview

What the UMN PhaseSpace setup is doing

The room uses PhaseSpace motion capture to track rigid objects instrumented with active LEDs. Once the tracker geometry is defined, the system estimates a live rigid-body pose and the ROS workflow publishes that motion into topics you can inspect, record, or use inside a robotics pipeline.

Location

Installed in Shepherd Labs

The system is already mounted in the Drone Lab, so the normal user flow starts with finding the room and getting the installed hardware online.

Tracker

LED geometry defines the rigid body

The marker arrangement on the object is what allows PhaseSpace to distinguish one rigid tracker from another.

Workflow

JSON is the handoff to ROS

Session profiles and Master Client are how you produce the tracker definition that the ROS node later loads.

ROS

Topics carry the motion data

The resulting pose can be published, visualized live, or recorded into bags for later analysis and mapping.
