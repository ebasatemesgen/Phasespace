Calibration

Calibrate only when the room setup changed.

Calibration and alignment are not normal daily startup steps. They matter when the room geometry changed, especially if cameras moved or the space was reconfigured. If the room is already tracking correctly, you usually do not need to recalibrate it.

Use carefully

Do not calibrate just because you are new to the room.

Calibration affects the shared system. Treat it as a corrective procedure, not as the default first step for normal use.

When It Is Needed

Common reasons to recalibrate

Cameras moved

Tracking geometry changed

If the camera placement changed, the room calibration can no longer be trusted as-is.

Alignment changed

World origin needs to be reset

If the floor origin or axis alignment must change, the alignment phase needs to be run again.

System health

Tracking quality dropped

If the room no longer tracks consistently after a physical change, recalibration may be necessary.

What You Need

Hardware and access for calibration

Hardware

Calibration wand

You need the calibration wand to collect the capture data used by the room calibration tool.

Hardware

Microdriver

The microdriver powers the wand LEDs during calibration and alignment.

Software

Configuration Manager access

The calibration and alignment tools live inside the Web Clients area of the Configuration Manager.

Calibration Sequence

The room calibration steps

1

Connect the wand hardware

Attach the microdriver to the calibration wand and power it on so the LEDs activate.

2

Open the calibration client

Go to the Configuration Manager, open Web Clients, then enter the calibration tool and connect.

3

Capture wand motion through the room

Wave the wand through the volume until the calibration client shows good enough camera coverage.

4

Calibrate and save

Finish the calibration inside the client and save it before disconnecting.

Alignment

Set the world origin after calibration

1

Place the wand at the intended origin

Use the room’s intended floor origin as the first reference point.

2

Record origin, +x, and +z

Those three snapshots define the aligned world frame used by the system afterward.

3

Save only if the alignment is correct

If the result looks wrong, reset and repeat instead of saving a questionable alignment.

Page goal

This page explains when calibration is necessary and how to run it without turning normal room use into a calibration task.
