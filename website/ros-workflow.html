ROS Support

What is supported in this project

ROS 1

Noetic

The ROS 1 package is intended for ROS Noetic workflows. It publishes tracked objects to /tf and /phasespace/rigids, with marker data available on /phasespace/markers.

Open ROS 1 README
ROS 2

Works with current ROS 2 distros

The ROS 2 workflow is the main path in this guide. The ROS 2 publisher connects to the PhaseSpace server, loads tracker JSON files, and publishes /phasespace and /phasespace_vel. In practice, this is the flexible option for current ROS 2 environments such as Foxy, Galactic, Humble, Jazzy, and similar setups.

What The Workflow Does

The operational sequence

1

Connects to the server

The node talks to the running PhaseSpace system through the device address you provide.

2

Loads the tracker definition

The JSON file tells the system which rigid body layout to create and stream.

3

Publishes pose topics

The tracked object becomes a live ROS pose stream that other robotics software can subscribe to.

4

Supports replay and analysis

That output can be visualized live, recorded into a bag, and turned into readable maps afterward.

Topics And Frames

What comes out of the ROS 2 workflow

Topic

/phasespace

Published as geometry_msgs/msg/TransformStamped for rigid-body pose data.

Topic

/phasespace_vel

Published as geometry_msgs/msg/TwistStamped for estimated linear velocity.

Frames

map and tracker IDs

The parent frame is map. Each rigid body appears with its own tracker ID as the child frame.

Mapping And Visualization

What happens after you record data

The mapping functionality lives in the visualization tools. After recording a bag, the offline renderer can turn the trajectory into a top-down 2D map or a 3D room-relative view. The same folder also includes a live viewer for the ROS 2 topic, so you can check the motion as it streams.

Live

Monitor the topic in real time

The live visualizer subscribes to the ROS 2 topic and shows the path in 2D or 3D while the system is running.

Offline

Render 2D and 3D trajectory maps

The bag visualizer reads recorded MCAP data and creates trajectory map outputs you can inspect or share.

Choosing a path

If you are starting fresh, use ROS 2 first.

ROS 1 support is still present for Noetic-based setups, but the clearest and most current route through this project is the ROS 2 workflow plus the visualization tools.
