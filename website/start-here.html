Startup Sequence

The normal order of operations

Follow this order and you avoid doing ROS or tracker work before the room itself is ready.

1

Find the hub and power it on

The hub is the physical starting point. If it is off, nothing else in the workflow matters.

2

Join the network

Use the room network path expected by the setup before opening the browser tools.

3

Open the Configuration Manager

Go to http://cs-phasespace.cs.umn.edu. The current guide lists admin and phasespace as the room credentials.

4

Confirm the hardware appears

Check the LED Devices tab and make sure the microdrivers you need are visible and active.

What you need physically

Hub, microdrivers, and LED hardware

The room hardware is already installed, but your tracked object still needs the appropriate tracker hardware attached.

What you need in software

Browser access before ROS

Do not jump straight to ROS commands. The browser-side room setup comes first, especially for confirming devices and profiles.

What comes next

Tracker setup

Once the room is online and your hardware is visible, the next job is to create or enable the tracker definition.

Ready to move on

You are done with this page when the room is reachable and the hardware is visible.

At that point, the next question is not where the room is. The next question is how to create a usable tracker definition for your object.

Page goal

This page is about location and startup. It gets the room ready before tracker or ROS work begins.
